## 二、核心学习内容

### 2.1 MCP 协议与服务器搭建

#### 2.1.1 MCP 核心原理
MCP（Model Context Protocol）是由 Anthropic 推出的开放协议，为 AI Agent 提供标准化的外部工具与数据调用接口，采用 Client-Server 架构，底层基于 JSON-RPC 2.0 规范通信。

**三大核心实体**：
- **Resources（资源）**：只读数据源，如本地文件、数据库查询结果、实时 API 数据流
- **Tools（工具）**：可执行函数，Agent 可主动调用并获取返回结果
- **Prompts（提示词模板）**：预定义指令模板，确保 Agent 遵循最佳实践执行任务

**传输层支持两种模式**：
- **Stdio 模式**：本地进程间通信，Client 启动 Server 作为子进程，适合本地开发调试
- **HTTP/WebSocket 模式**：网络远程访问，适用于生产环境部署与多客户端共享

#### 2.1.2 MCP 服务器开发实践

**技术栈选型**：Python + FastMCP SDK

**环境搭建步骤**：
```bash
# 创建虚拟环境
python -m venv mcp-env
source mcp-env/bin/activate

# 安装 MCP SDK
pip install mcp[cli] python-dotenv httpx
```

**最小可运行服务器示例**：
```python

mcp = FastMCP(name="knowledge-server", version="1.0.0")

@mcp.tool()
def search_knowledge(query: str) -> str:
    """
    在知识库中检索与查询相关的文档片段
    Args:
        query: 用户的检索问题
    Returns:
        相关的知识库内容
    """
    # 此处对接向量数据库检索逻辑
    return f"检索结果：关于'{query}'的相关知识..."

if __name__ == "__main__":
    mcp.run()
```

**客户端配置（以 Claude Desktop 为例）**：
在 `claude_desktop_config.json` 中添加服务器配置：
```json
{
  "mcpServers": {
    "knowledge-server": {
      "command": "python",
      "args": ["/path/to/knowledge_server.py"]
    }
  }
}
```

#### 2.1.3 生产级 MCP 服务器架构要点
- 采用 `uv` 替代传统 `pip` 管理依赖，提升安装速度与依赖解析精度
- 实现工具注册的类型安全，利用 Python 类型提示自动生成工具 Schema
- 加入错误处理与日志机制，通过 `loggingNotification` 向客户端推送运行状态
- 支持多传输层切换，本地开发用 Stdio，生产部署用 SSE 或 HTTP 模式

### 2.2 知识库检索系统（RAG）

#### 2.2.1 RAG 全流程架构
检索增强生成（RAG）是 Agent 知识库能力的核心实现，完整工作流分为**离线建库**与**在线检索**两大阶段：

```
原始文档 → 数据清洗降噪 → 智能文本分块 → Embedding 向量化 → 向量库索引存储
                                                          ↓
用户提问 → 问题向量化 → 相似度检索 → 重排序 → 上下文拼接 → LLM 生成回答
```

#### 2.2.2 关键技术要点

**数据预处理阶段**：
- **清洗降噪**：去除空行、水印、页码、特殊符号与重复段落
- **文本分块策略**：采用递归字符分块 + 重叠补全，推荐 `chunk_size=500-1000`，`chunk_overlap=50-100`，平衡检索精度与语义完整性
- **元数据提取**：记录文档来源、页码、作者等信息，用于回答溯源

**检索层核心组件**：

| 组件       | 作用                           | 主流选型                                        |
| ---------- | ------------------------------ | ----------------------------------------------- |
| 向量化模型 | 将文本转为高维语义向量         | BGE、OpenAI Embeddings、Sentence-BERT           |
| 向量数据库 | 存储向量并提供相似性检索       | 小规模：Chroma、FAISS；大规模：Milvus、Weaviate |
| 重排序模型 | 对初筛结果精排，提升召回准确率 | BGE-Reranker、CrossEncoder                      |

#### 2.2.3 Agentic RAG 进阶模式
区别于传统的"检索即生成"两阶段模式，Agentic RAG 让 LLM 自主决策是否检索、检索几次、如何组合多源信息，适用于复杂多跳问答场景。实现方式基于 ReAct 框架：
1. **Thought**：模型思考当前信息是否充足
2. **Action**：若信息不足，生成检索查询并调用搜索工具
3. **Observation**：获取检索结果
4. 循环迭代，直至信息充足后输出最终答案

### 2.3 Agent 网站整体架构设计

#### 2.3.1 三层架构模型
完整的 Agent Web 应用采用分层解耦设计，分为三大层级：

| 层级         | 核心职责                                           | 技术选型                                   |
| ------------ | -------------------------------------------------- | ------------------------------------------ |
| 前端 UI 层   | 用户交互、消息流展示、工具调用可视化、任务状态追踪 | Next.js / Vue3 + TypeScript + Tailwind CSS |
| 后端 API 层  | 请求鉴权、会话管理、服务编排、流式输出、审计日志   | FastAPI / Spring Boot + WebSocket/SSE      |
| Agent 核心层 | 推理规划、工具调用、RAG 检索、记忆管理             | LangGraph / LlamaIndex / AutoGen           |

#### 2.3.2 前端核心功能模块
- **对话交互界面**：支持流式输出、Markdown 渲染、代码高亮
- **工具调用可视化**：实时展示 Agent 调用了哪些工具、参数是什么、返回结果如何
- **知识库管理**：文档上传、分块预览、向量索引状态监控
- **会话历史**：多会话管理、消息检索、上下文清理

#### 2.3.3 后端关键设计
- **流式响应**：使用 SSE（Server-Sent Events）实现打字机效果，提升用户体验
- **会话持久化**：数据库存储对话记录与 Agent 运行轨迹，支持中断恢复
- **并发控制**：限制单用户并发请求数，防止 LLM API 超限
- **错误重试**：针对 LLM 调用失败、工具超时等场景实现降级与重试机制

## 四、问题与解决方案

### 4.1 MCP 服务器调试困难
**问题**：Stdio 模式下无法直观查看请求响应日志，排错效率低。
**解决**：使用 MCP Inspector 工具进行可视化调试，或在代码中启用 `debug=True` 模式，将协议交互日志输出到文件。

### 4.2 长文档检索准确率不足
**问题**：技术文档分块后，跨段落的连贯知识被割裂，多跳问题回答质量下降。
**解决**：引入父子分块策略，检索时匹配小块，返回时加载父块完整上下文；同时加入重排序模型对初筛结果二次精排。

### 4.3 前端流式渲染卡顿
**问题**：Agent 调用多轮工具时，前端频繁重渲染导致页面卡顿。
**解决**：采用虚拟列表优化长对话渲染，将工具调用详情默认折叠，按需展开；使用 React 19 的 useOptimistic 减少状态更新抖动。

## 六、总结与思考

本周的学习清晰了 Agent 网站搭建的完整技术路径：MCP 解决了工具标准化接入的问题，RAG 赋予了 Agent 外部知识能力，而前后端分层架构则保障了系统的可扩展性与用户体验。三者并非孤立存在，而是可以深度融合——例如将 RAG 检索能力封装为 MCP 工具，供上层 Agent 统一调用，形成标准化的能力复用体系。

后续学习的重点将从"单点技术验证"转向"系统整合优化"，重点关注多工具协同的编排逻辑、检索效果的量化评估，以及生产环境下的性能与稳定性保障。
